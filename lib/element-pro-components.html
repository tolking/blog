element-pro-components 是一个基于 element-plus (opens new window), vue (opens new window), vue-router (opens new window) 的一个适用于中后台快速开发的一套组件库

# 起因

在工作中需要开发及维护多个后台管理系统。经常在想如何更优雅的处理不同项目间的协同开发，尝试过 git单模板多项目的探索 的方法优化，但还是觉得不够彻底。增加公共功能和修复公共的BUG，都需要前往 template 项目修改，完成后手动同步到各个需要的项目。后来尝试将公共的组件分离到一个单独的本地仓库中，通过 npm 能够安装本地包的特点，在需要的项目中安装。这样安装、修改、更新比起 template 都要方便不少，于是就有了做这个项目的想法。

当然这也仅仅是一个自用顺带分享的项目，不太可能像一些模版项目完善。它的目标不是代替模版项目，而是提供一套简单的组件任你自由发挥制作一套属于自己的模版项目

# 优势

  • 通过更新依赖版本及可同步组件变动，方便使用
  • 不需要花时间精简模版中未使用的依赖
  • 支持按需引入组件及样式
  • api 风格贴近 element-plus
  • 基于 TypeScript 开发，完善的类型推断
  • 与业务完全脱离，避免不同业务的组件混用产生错误
  • 提供完善的代码提示

# 组件

# 布局相关

  • Layout (opens new window) 基础的布局组件，能够自动从 vue-router 获取路由信息生成菜单
  • Menu (opens new window) 自动从 vue-router 获取路由信息生成菜单，已包含在 Layout 组件中
  • Breadcrumb (opens new window) 根据当前页面路由信息自动生成面包屑
  • Tabs (opens new window) 记录浏览的历史页面自动生成 tabs
  • Link (opens new window) 方便处理内部和外部跳转

# 表单相关

  • Form (opens new window) 根据 columns 配置信息动态生成表单，具体使用请查看文档
  • Radio (opens new window) 根据 data 信息生成单选组，方便在 Form 中使用单选
  • RadioButton (opens new window) 同 Radio
  • Checkbox (opens new window) 根据 data 信息生成多选组，方便在 Form 中使用多选
  • CheckboxButton (opens new window) 同 Checkbox
  • Select (opens new window) 根据 data 信息生成选择器，方便在 Form 中使用选择器
  • TreeSelect (opens new window) 树形结构选择器
  • InputTag (opens new window) 多标签输入框
  • AutocompleteTag (opens new window) 同 InputTag，额外提供输入建议

# 表格相关

  • Table (opens new window) 根据 columns 配置信息动态生成表格，具体使用请查看文档
  • Crud (opens new window) 基于 Form 和 Table 用于快速构建增删改查页面
  • ColumnSetting (opens new window) 配合 Table 或 Crud 使用，动态控制表格中某一项的显示与隐藏

# 其它

  • Descriptions (opens new window) 根据 columns 配置信息动态列表形式的展示字段，具体使用请查看文档
